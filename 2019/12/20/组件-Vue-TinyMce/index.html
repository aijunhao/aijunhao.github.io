组件-Vue-TinyMce富文本组件

前言

本文讲述的是 TinyMce 富文本组件的应用，相对来说，这个富文本组件比较好看，并且配置相对简单，最关键的是功能强大。

其他常见的富文本编辑器有：

UEditor、Kindeditor、Simditor、CKEditor、wangEditor、Suneditor、froala等。

TinyMce 对 Vue 进行了集成，可以直接使用 @tinymce/tinymce-vue 这个插件，详细见：https://www.tiny.cloud/docs/integrations/vue/ ，但是这样需要使用官方的 API-KEY，有必要的话可以去申请，然后按照教程进行配置，会非常简单。

当不去申请 API-KEY ，就需要自己对 TinyMce 进行适配了，本文将详细介绍。

环境配置

当前使用环境为 vue-cli 3 ，tinymce 5.0.12 ，@tinymce/tinymce-vue 2.1.0 。

简介与入门

选择器配置

selector 是 TinyMCE 的重要配置选项，使用 CSS 选择器语法来确定页面上哪个元素被 TinyMCE 替换成编辑器。

在经典模式中，TinyMCE 将会在 selector 指定的 textarea 处插入一个 iframe 。

而在内联模式下，此选项可用于任何块元素，内联模式直接将编辑器插入到指定的容器中，而不使用 iframe ，再次提示，此处css样式将继承自外部。

模式

经典模式

默认的就是经典模式，和普通的富文本编辑器基本一致。

内联模式

内联模式对提升用户体验相当有用，页面的编辑视图与阅读视图合二为一，提供了无缝的体验，实现真正的所见即所得！

在内联编辑模式下，编辑器不会将selector指定的元素替换为iframe，而是直接将编辑器组件融入其中。

注意：selector的目标设为textarea是无法启用内联模式的，必须是普通的块元素，div最佳。

指定参数 inline:true 将开启内联模式：

沉浸模式

沉浸模式类似于内联模式，只要将 toolbar 设为 false ，内联模式就能变成沉浸模式。在该模式下，编辑器以轻巧的方式呈现，此模式提供了将常用的链接、图像、表格等以快速插入的方式。

插件

使用插件有几个通用步奏：

  1. 导包：在 vue 中自己配置的 TinyMce 都要自己导包。
  2. 在 plugins 插件参数中启用插件。
  3. 在 toolbar 工具栏参数中显示工具按钮。

其他特殊插件需要进行配置。

tinymce-vue 介绍

因为我们不使用 API-KEY 时使用 TinyMce需要额外配置，而最终来说还是基于 @tinymce/tinymce-vue 这个插件的，只是不再需要官方的 TinyMce 配置了。

这个插件组件接受如下属性：

api-key

如果没有 API-KEY，自己配置，就不需要填写这个属性了。

disabled

设置编辑器为只读模式或者可编辑模式。

id

编辑器的id。用于使用 tinymce.get（'ID'） 方法检索编辑器实例。默认为自动生成的UUID。

init

初始化配置，一个对象类型，会发送到编辑器的 tinymce.init() 方法中进行配置。

initial-value

编辑器的初始内容。

inline

用来设置编辑器的内联模式。

model-events

设置 v-model 的触发事件，默认为：change keyup undo redo 。

plugins

是 init 中插件部分属性的设置，插件属性可以在 plugins 中设置，也可以在 init 中设置。

tag-name

只有在编辑器为内联模式时有效，用来设置默认的内联模式编辑器的 html 标签。

toolbar

是 init 中工具栏部分属性的设置，工具栏属性可以在 toolbar中设置，也可以在 init 中设置。

v-model

双向数据绑定，绑定编辑器内容（ html 格式）。
