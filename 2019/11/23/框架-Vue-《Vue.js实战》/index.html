前言

此文章内容是《Vue.js实战》的读书笔记。

作者是梁灏，内容基于 Vue.js 2.0 。是 Vue.js 框架作者尤雨溪写推荐序的一本书。

此外，本书作者也是开源组件库 iView 的作者。

《Vue.js实战》分三个部分，”基础篇”，“进阶篇” 和 “实战篇”。

“基础篇”包括：

  • 双向绑定数据；
  • 计算属性；
  • 内置指令和自定义指令；
  • 组件等。

“进阶篇”包括：

  • Render 函数；
  • webpack 的使用；
  • Vue.js 插件

”实战篇“包括：

  • 两个 iView 组件的设计和实现思路；
  • 两个完整的实战项目。

此次学习的主要目的在于 ”进阶篇“ 和 ”实战篇“，通过本书的案例对比我已经写过的项目，来进一步提升我的项目开发水平。同时通过 ”基础篇“ 进行查漏补缺，“基础篇” 会简略带过。

基础篇 - 第 3 章 计算属性

在一个计算属性里可以完成各种复杂的逻辑，包括运算，函数等，最要最终返回一个结果就可以。

每一个计算属性都包含一个 getter 和一个 setter ，一般只用默认的 getter 就行了。

在实际开发中，计算属性的应用如下：

  • 动态计算元素的样式名称 class 或者内联样式 style。
  • 组件间动态传递 props

计算属性与 methods

计算属性对象 conputed 和方法对象 methods 主要有两个区别：

  1. 计算属性对象 conputed 中的方法不能传参，而方法对象 methods 可以。
  2. 计算属性是基于依赖缓存的，当依赖的数据发生变化的时候才会重新计算，不然就直接从缓存中拿数据返回。而方法对象每调用一次就会重新执行一次。

基础篇 - 第 5 章 内置指令

Vue.js 中的指令是带有特殊前缀v- 的 HTML 特性，绑定一个表达式，将一些特性应用到 DOM 上。

v-cloak

在插值表达式中，如 ，当网速比较慢时，Vue.js 文件还没有被加载完成时，这就是一个普通字符串，在页面中显示 字样。等Vue 加载完成，创建实例，编译模板后，DOM 才会被替换，所以在这个过程中屏幕是由闪动的。

v-clock 适用于简单项目的页面闪动。

v-once

v-once 作用时定义它的元素或组件包括其子节点只渲染一次，之后相当于静态页面，页面不再随数据变化重新渲染。

v-if 和 v-else

如上是一个简单的 v-if 和 v-else 的案例。

在输入内容之后点击按钮进行切换，虽然 DOM 变了，但是之前在输入框键入的内容没有被改变，只是替换了 placeholder 内容，这说明 input 标签被复用了。

Vue 在渲染元素时，处于效率考虑，会尽可能地复用已有的元素而非重新渲染。

如果不希望元素被复用，可以给元素添加 key 属性，属性值必须是唯一的。

指令修饰符

  • .stop: 阻止冒泡。
  • .prevent: 不再重载页面。
  • .capture: 事件捕捉。
  • .self: 事件只在该元素本身触发时回调。
  • .once: 只触发一次。

基础篇 - 第 7 章 组件详情（重点）

组件的功能就是将某个具体功能代码封装成组件，解决复用的问题。

组件的 data 必须是一个函数，否则会导致数据污染，在工程化中甚至会直接报错。

此外就是组件传参问题，父组件通过 props 将属性传入子组件，子组件通过 $emit(name, value) 自定义事件将数据传入到父组件并使用 @name="fun(value)" 的形式接收，或者 $on(value, fun(value))。

在子组件中接收 props 时，最好明确规定传入的类型，是否必须传入值，如果有必要可以设置默认值和验证函数。如果传入的值不能直接使用，可以用 computed 计算属性进行转换。

非父子组件通信

在非父子组件中如果需要实现通信，可以使用如下几种方法：
  • bus 中央事件总线：适合小型项目。
  • vuex 全局状态管理工具：适合大型项目。
  • this.$parent this.$children 父链和子组件索引：在子组件中使用this.$parent 可以访问该组件的父实例或组件，父组件可以通过 this.$children 访问子组件。父链和子组件有一个特点，就是可以向上或者向下无限递归访问，直到跟实例或最内层的组件。
  • localStrage 本地存储：将字段存储在本地，可以实现永久存储，任意地方访问。
  • sessionStrage ：将字段存在浏览器，在浏览器关闭前有效。

在业务中，应尽量避免父子组件数据依赖，减少父子组件耦合。

组件通信内容将在单独模块介绍，见《框架-Vue-组件通信》

slot 插槽

插槽就是一个萝卜一个坑，子组件挖坑（slot插槽），父组件填萝卜（内容）；具名插槽就是把萝卜和坑标了名称，只能对号入座，最后多的无名萝卜进无名的坑（默认插槽），没有无名的坑就丢弃萝卜，坑里有本来就有萝卜那就把萝卜拔出来填父组件的萝卜。

具名插槽

具名插槽的写法在最新的 Vue 官方文档中有些更新。

在子组件中，具名插槽在 slot 中用 name 属性指定，而在父组件中，可以用 name 属性或者 class 属性或者 v-slot 指令指定。

并且在使用上一般建议在父组件中使用 template 标签并结合 v-slot 指令使用。

作用域插槽

作用域插槽的目的是让父组件中的插槽内容能够访问子组件中的数据。

如上，是使用作用域插槽将子组件数据提供给父组件的例子。在 vue 2.6.0 之后，废弃了 slot-scope 的语法，使用 v-slot 来定义插槽 prop 的对象名称，并且允许不同的插槽使用同一个名字。

在子组件中，用 v-bind 绑定子组件数据，如 :outValue="value" 左侧为传出的名称，右侧为子组件中的数据。然后在父组件中 v-slot 后接具名插槽名称 :default 然后定义在父组件中使用的名称 headerProps ，之后就能在当前子组件作用域范围内使用了（在 <child></child> 中使用有效，在外面使用会报错）。

访问 slot

用来访问被插槽分发的内容。每个具名插槽有其相应的属性 (例如：v-slot:foo 中的内容将会在 vm.$slots.foo 中被找到)。default 属性包括了所有没有被包含在具名插槽中的节点，或 v-slot:default 的内容。

平常时候不会使用 $slots 在使用渲染函数书写一个组件时，访问 vm.$slots 最有帮助。

递归组件
